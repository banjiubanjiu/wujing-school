# 三、工作计划

## P0 平台底座

**目标**：能登录、能区分学员/教师/教务、每个角色有自己的首页和菜单。

**要做的功能块**

- 账号体系：用户、角色、组织（学院/大队/中队/班级）。
- 登录登出：账号密码登录、会话保持。
- 权限壳子：
  - 根据角色进入学员端 / 教师端 / 管理端首页。
  - 每个角色看到自己的菜单（页面内容可先占位）。

**验收标准**

- 用不同账号登录，能进入不同的首页和菜单。
- 未登录访问业务页面，会被要求登录。

## P1 教务闭环（课堂教学线）

**目标**：真实支撑一个学期的“课堂教学”：**入学建档 → 排课上课 → 考试 → 查成绩**。

**要做的功能块**

1. 学籍 & 班级
   - 学员档案：导入、查询、编辑、导出。
   - 学籍状态：在读 / 休学 / 退学 / 毕业（含异动记录）。
   - 班级/中队信息：年级、专业、所属单位、班主任。
2. 专业 / 课程 / 培养方案（基础）
   - 专业列表。
   - 课程列表（代码、名称、学分、类别）。
   - 培养方案：为“专业+入学年份”配置应修课程清单。
3. 开课 & 排课 & 课表
   - 教学班：学期+课程+任课教师+教学对象（班级/中队）。
   - 排课：为教学班安排上课时间/地点，简单冲突检查（教师/教室）。
   - 课表：
     - 学员“我的课表”。
     - 教师“我的课表”。
     - 管理端按班级/教师/教室查看课表。
4. 成绩管理（含基础考务）
   - 考试/成绩记录：平时/期中/期末，总评。
   - 老师录入、修改成绩。
   - 教务审核/发布成绩。
   - 学员按学期查看成绩单。

**验收标准**

- 新建一批学生 → 分配专业和班级。
- 配置课程和教学班 → 排课 → 学员/教师能看到自己的课表。
- 为某门课录入成绩 → 教务审核 → 学员在成绩页面看到结果。

## P2 训练闭环（实训 / 警务训练）

**目标**：完整跑通“训练线”：**训练项目 → 训练任务 → 成绩 → 达标统计**，并管理场地与装备。

**要做的功能块**

1. 训练项目与标准
   - 训练项目字典：体能、射击、队列等。
   - 每个项目的计量方式和达标标准。
2. 训练计划与任务
   - 按学期/阶段，为年级/中队制定训练计划。
   - 生成具体训练任务（时间、地点、项目、对象）。
3. 训练成绩 & 统计
   - 教官批量录入训练成绩。
   - 系统判定是否达标。
   - 学员查看个人训练记录。
   - 管理端按班级/中队/项目统计达标率。
4. 场地与装备
   - 场地管理：实训场地台账、使用安排。
   - 装备管理：装备台账、借用归还、报修记录。

**验收标准**

- 为某中队制定训练计划 → 生成训练任务 → 教官录入成绩 → 学员看到自己的训练记录。
- 在“训练统计”里能看到某年级/中队的达标率。
- 能查询某天某个场地/装备的使用和借出情况。

## P3 教学空间（学员 & 教师）＋ 选课 ＋ 资源

**目标**：让师生“日常教学活动”都发生在系统里，而不只是后台管理。

**要做的功能块**

1. 学员学习空间
   - 学员首页：今日课程、近期考试、待办（作业/评教/训练）、最新通知。
   - 作业：
     - 学员查看课程作业、截止时间。
     - 在线提交作业，查看批改结果和评语。
   - 成绩 & 训练总览：一个入口看到自己的学业和训练情况。
2. 教师教学工作台
   - 教师首页：今日授课、待录入成绩、待批改作业。
   - 授课班级 & 学生名单。
   - 作业布置与批改：
     - 为课程布置作业。
     - 查看提交情况、打分和点评。
   - 教学资源上传：按课程/教学班上传课件、案例等。
3. 选课管理（选修课）
   - 选修课计划：时间窗口、容量、适用年级。
   - 学员选课/退课。
   - 管理端查看选课统计和名单。
4. 教学资源/文献库
   - 内部文献/案例/规章库。
   - 分类、标签、检索。
   - 按角色/单位控制可见范围。

**验收标准**

- 学员登录 → 在首页看到今天课表、作业、通知 → 进入课程提交作业。
- 教师登录 → 在工作台看到自己的课程、作业布置/批改入口。
- 一次完整的选修课选课流程可以在系统内完成并统计。

## P4 教学质量 & 心理健康 & 决策驾驶舱

**目标**：从“操作系统”升级为“管理决策与学生支持系统”。

**要做的功能块**

1. 教学评价（评教）
   - 评价模板：问卷配置（学生评教/教师自评/同行/督导）。
   - 评教任务：按学期/课程/教师发起。
   - 线上填表。
   - 结果分析：按课程/教师/班级汇总和对比。
2. 心理健康与预警
   - 心理测评任务，下发量表。
   - 学员在线填写。
   - 系统计算结果，辅助标记风险等级。
   - 心理老师维护重点关注名单和记录。
3. 决策驾驶舱（非 AI）
   - 汇总指标：
     - 成绩通过/补考/重修情况。
     - 训练达标率。
     - 教学评价结果。
     - 心理高风险人数。
   - 多维统计：按年级/专业/大队/中队查看。
   - 做成一个管理端“驾驶舱”大屏。

**验收标准**

- 学期末能发起评教任务 → 学员/教师在线完成 → 教务查看评教结果。
- 心理老师能看到近期测评、关注名单。
- 领导登录 → 在驾驶舱看到本学期教学、训练、评教、心理的关键指标。

## P5 AI & 数字人

**目标**：在完整业务数据基础上，用 AI 和数字人提升学习、教学和决策体验。

**要做的功能块**

1. 学员 AI 学习助手
   - 在学员端提供 AI 对话：问知识点、问学习建议。
   - 结合学员成绩/训练情况，给个性化提醒（易挂科课程、训练短板等）。
2. 教师 AI 教研助手
   - 辅助生成教学提纲、课堂问题、训练案例。
   - 分析评教结果和教学数据，给优化建议。
3. 智能问数（自然语言查数据）
   - 管理者用自然语言提问（如“本学期大一体能不达标的有多少？”）。
   - 系统返回数字+简单图表。
   - 常用问题可以保存为“智能报表模板”。
4. 数字人教官 / 数字辅导员
   - 在指定场景嵌入数字人（迎新、训练说明、心理宣传等）。
   - 数字人可以播放预设脚本，也可以回答简单问题（接 AI）。

**验收标准**

- 学员在学习空间中可以和 AI 助手互动，得到有用的学习/训练建议。
- 教师可以用 AI 快速生成教学内容草案。
- 领导可以通过自然语言问出一两个关键指标，系统给出图表答案。
- 至少有 1–2 个场景有可演示的数字人（比如“新生入学引导”或“训练准备须知”）。
